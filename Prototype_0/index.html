Header prototype

Above is a cross-browser (I hope; only tested in IE6, FF3.6.3, Opera, and Chrome), semantic header. I picked up the classing from Drupal. Branches have children, leaves do not. Not sure if the second tier will appear on the hover state of a first tier element, or whether it will appear only on the active state of a first tier element.

I'm fairly sure this prototype will explode in IE7/8.

The design

Right now this prototype is a rough approximation to the wireframes since I used the existing VML prototype as the basis for design. This prototype's purpose is to demonstrate a cross-browser, semantic navigation system within the 960.gs framework.

The third tier

The third tier isn't a proposal nor is it an assumption that the business requirements call for it. It's just a result of my curiosity.

Second tier hover states in IE6

Since we know the height of the second tier, we will easily be able to set the height of the hover image in Photoshop or whatever so that we can position the arrow. And since the widths of each of the second tier navigation elements aren't extremely different, we could find a "happy medium" or average width for the image to simulate center alignment. If that's too hokey, we could just switch out the PNG with a GIF.
